# Schedule — 工作计划管理技能

> SCHEDULE.md 是你的工作记忆。框架会自动将当前计划注入到对话上下文中，
> 你只需在合适时机**使用工具更新它**。

## 你需要做的

### 收到大型需求时 → 拆解并录入

1. 将需求拆分成独立的子任务
2. 为每个子任务评估优先级 (P0/P1/P2)
3. 调用 `schedule_add_task` 逐条插入
4. 向用户确认优先级和顺序
5. 从最高优先级开始逐个击破

### 完成一个任务后 → 立即标记

调用 `schedule_complete`，附上简要总结。不要拖延。

### 工作中发现新问题 → 即时插入

1. 评估优先级
2. 调用 `schedule_add_task` 插入
3. 告知用户："发现了 xxx，已加入计划，优先级 P1"
4. 继续当前任务（除非新问题是 P0 阻塞性的）

### 用户问进度 / P0 完成时 → 汇报

调用 `schedule_review` 生成进度报告。

## 优先级规则

| 级别 | 含义 | 处理原则 |
|------|------|----------|
| P0 🔴 | 阻塞性/紧急 | 立即处理，不可被打断 |
| P1 🟡 | 重要但不紧急 | 当前 P0 完成后按序执行 |
| P2 🟢 | 一般/可延后 | 有空再做，不主动提起 |

## 注意

- 框架已自动在对话中注入当前 SCHEDULE.md 的待办摘要，你不需要主动 `schedule_load`（除非需要完整细节）
- 不要在 SCHEDULE.md 中记录技术细节，只记录任务名 + 简要备注
- 单个文件最多保留 50 条已完成记录，超出时用 `schedule_archive` 归档
