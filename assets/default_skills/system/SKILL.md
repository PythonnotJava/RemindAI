# System — 开发环境探测技能

> 框架会在会话开始时自动探测系统环境并注入上下文。
> 你只需在特殊情况下手动调用工具。

## 自动行为

会话首轮，框架已自动执行 `system_probe(category: all)` 并将结果注入。
你可以直接在上下文中看到系统已安装的工具列表，无需手动调用。

## 何时手动调用

### system_probe

以下情况手动调用：
- 需要探测特定工具（`category: custom`, `tools: ['ffmpeg', 'magick']`）
- 用户中途安装了新工具，需要刷新探测结果

### system_env

以下情况调用：
- 需要获取特定环境变量的值（如 `JAVA_HOME`、`GOPATH`）
- 需要确认代理配置（`HTTP_PROXY`）
- 用户问"我的环境变量怎么设的"

## 使用原则

1. **根据已知环境选择方案** — 有 pnpm 用 pnpm，有 rg 用 rg，无 docker 不建议容器化
2. **缺失工具时建议安装** — 明确告知用户缺什么、怎么装
3. **不重复探测** — 上下文里已经有了，别再调一遍

## 边界

- 只读操作，不修改系统环境
- 敏感变量（含 KEY/SECRET/TOKEN/PASSWORD）只报告"已设置"，不返回值
- 每次探测超时 3 秒
